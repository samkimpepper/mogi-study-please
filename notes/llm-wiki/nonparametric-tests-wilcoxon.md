# 비모수 검정과 Wilcoxon 기초

> [!summary]
> 비모수 검정은 데이터가 정규분포라고 보기 애매할 때 쓰는 통계 검정이다.
>
> LLM wiki 검색 평가에서 Wilcoxon signed-rank test가 나오면, 보통 같은 gold task를 기존 검색기와 새 검색기가 같이 풀었고 그 task별 차이를 비교했다는 뜻이다.

## 왜 봐야 하나

LLM wiki 검색 PR에서 이런 문장이 나올 수 있다.

```text
We used a Wilcoxon signed-rank test on paired task scores.
```

이 말은 대략 다음 뜻이다.

```text
같은 문제들을 기존 검색기와 새 검색기가 모두 풀게 했다.
각 문제별 점수 차이를 봤다.
점수 차이가 정규분포라고 보기 애매해서 Wilcoxon 검정을 썼다.
```

> [!info]
> 백엔드 관점에서 보면 같은 요청 목록을 기존 서버와 새 서버에 각각 보내고, 요청별 응답 시간 차이를 비교하는 것과 비슷하다.

## Paired Test

`paired test`는 같은 대상에 대해 두 조건을 비교하는 테스트다.

LLM wiki 검색 평가에서는 다음 상황이 paired test에 해당한다.

```text
gold task 17개가 있다.

각 task를:
- 기존 검색기
- 새 검색기

둘 다 풀게 한다.
```

예를 들면 다음과 같다.

| Task | 기존 검색기 점수 | 새 검색기 점수 |
| --- | ---: | ---: |
| 1 | 0 | 1 |
| 2 | 1 | 1 |
| 3 | 0 | 0 |
| 4 | 0.5 | 1 |

여기서 task 1의 기존 검색기 점수와 task 1의 새 검색기 점수는 서로 묶여 있다.

둘 다 같은 문제를 푼 결과이기 때문이다.

따라서 비교 방식은 다음에 가깝다.

```text
기존 검색기 전체 평균 vs 새 검색기 전체 평균
```

만 보는 것이 아니라,

```text
Task 1에서 old vs new
Task 2에서 old vs new
Task 3에서 old vs new
...
```

처럼 같은 문제 안에서 차이를 보는 것이다.

> [!summary]
> 같은 17개 문제를 두 검색기가 같이 풀었으니 paired다.
>
> 문제마다 난이도가 다르기 때문에, 같은 문제끼리 묶어서 비교해야 공정하다.

## 왜 paired가 중요한가

만약 기존 검색기는 어려운 문제 17개를 풀고, 새 검색기는 쉬운 문제 17개를 풀었다고 하자.

```text
기존 검색기: 어려운 문제 17개
새 검색기: 쉬운 문제 17개
```

이러면 새 검색기 점수가 좋아 보여도 진짜 검색기가 좋아진 것인지 알 수 없다.

문제 난이도가 다르기 때문이다.

같은 gold task를 둘 다 풀게 하면 문제 난이도를 어느 정도 통제할 수 있다.

```text
Task 1: 같은 문제에서 old와 new 비교
Task 2: 같은 문제에서 old와 new 비교
Task 3: 같은 문제에서 old와 new 비교
```

> [!example]
> 같은 API 요청 17개를 기존 서버와 새 서버에 각각 보내고 응답 속도 차이를 보는 것과 비슷하다.
>
> ```text
> request 1: old 120ms, new 90ms
> request 2: old 80ms, new 85ms
> request 3: old 300ms, new 180ms
> ```
>
> 이때도 요청별로 묶어서 비교해야 한다.

## Wilcoxon Signed-Rank Test

`Wilcoxon signed-rank test`는 paired 데이터에서 두 방식의 차이가 한쪽으로 일관되게 치우쳐 있는지 보는 비모수 검정이다.

LLM wiki 검색 평가에서는 다음 질문을 보는 데 가깝다.

```text
같은 17개 문제를 봤을 때,
새 검색기가 기존 검색기보다 일관되게 더 나은가?
```

절차를 단순화하면 다음과 같다.

```text
1. 각 task마다 새 점수 - 기존 점수를 계산한다.
2. 차이가 0인 것은 보통 제외하거나 따로 처리한다.
3. 차이의 크기 순위를 매긴다.
4. + 방향 차이가 큰지, - 방향 차이가 큰지 본다.
```

예를 들어 다음과 같은 차이가 있을 수 있다.

```text
Task 1: 새 방식이 +0.3 좋음
Task 2: 새 방식이 +0.1 좋음
Task 3: 차이 없음
Task 4: 새 방식이 -0.2 나쁨
Task 5: 새 방식이 +0.4 좋음
```

Wilcoxon은 단순히 평균만 보는 것이 아니라, task별 차이의 방향과 순위를 본다.

> [!info]
> 직관적으로는 "대부분의 문제에서 새 방식이 더 좋아지는 경향이 있는가?"를 보는 검정이라고 이해하면 된다.

## 왜 t-test 말고 Wilcoxon을 쓰나

`t-test`는 보통 데이터의 차이값이 정규분포에 가깝다는 가정을 둔다.

paired t-test라면 각 문제별 차이값이 대략 정규분포를 따른다고 보는 쪽에 가깝다.

```text
차이값 = 새 검색기 점수 - 기존 검색기 점수
```

하지만 gold task 17개로 검색 평가를 할 때는 이 가정이 애매할 수 있다.

이유는 다음과 같다.

```text
1. 샘플이 17개로 작다.
2. 점수가 0, 1 또는 0.25, 0.5처럼 듬성듬성할 수 있다.
3. Recall@5 같은 지표는 0~1 범위 안에 갇혀 있다.
4. 문제마다 난이도가 다르다.
5. 차이값이 예쁜 종 모양으로 퍼진다고 보기 어렵다.
```

예를 들어 task별 Recall@5가 다음처럼 나올 수 있다.

```text
0
0.25
0.5
0.75
1
```

이 값들은 키나 몸무게처럼 자연스럽게 연속적인 측정값이라기보다, 정답 문서 몇 개를 찾았는지에서 나온 비율에 가깝다.

그래서 정규분포라고 강하게 가정하기 어렵다.

> [!warning]
> task 수가 작고 점수가 0~1 사이의 듬성듬성한 값이면, "차이값이 정규분포다"라고 말하기 조심스럽다.

## 정규분포라고 보기 애매하다는 뜻

정규분포는 대략 다음과 같은 모양을 말한다.

```text
대부분은 평균 근처에 몰림
양쪽으로 갈수록 적어짐
종 모양으로 퍼짐
```

하지만 검색 평가 점수는 보통 이런 성질을 가진다.

```text
0점 아니면 1점처럼 튀는 값이 많다.
0과 1 사이에 갇혀 있다.
문제 수가 적으면 분포 모양을 판단하기 어렵다.
각 문제 난이도에 따라 차이가 들쭉날쭉하다.
```

17개 task면 더더욱 분포 모양을 확신하기 어렵다.

그래서 PR 작성자가 Wilcoxon을 썼다면 보통 다음 판단일 가능성이 높다.

```text
같은 gold task를 두 검색기가 같이 풀었으니 paired 비교다.
task 수가 적고 점수 분포가 정규분포라고 보기 어렵다.
따라서 paired t-test보다 Wilcoxon signed-rank test를 쓰자.
```

## t-test와 Wilcoxon 비교

| 구분 | t-test | Wilcoxon signed-rank test |
| --- | --- | --- |
| 비교 대상 | 평균 차이 | 차이의 방향과 순위 |
| 데이터 가정 | 차이값이 정규분포에 가깝다고 봄 | 정규분포 가정을 덜 함 |
| LLM wiki 17 task | 가정이 애매할 수 있음 | 더 보수적으로 쓰기 좋음 |
| 핵심 질문 | 평균적으로 얼마나 좋아졌나 | 일관되게 한쪽이 나은가 |

> [!tip]
> 짧게 말하면:
>
> 같은 17개 문제를 두 검색기가 같이 풀었으니 paired test이고, 점수가 정규분포라고 보기 애매하니 Wilcoxon signed-rank test를 쓴 것이다.

## PR에서 읽는 법

PR에 다음 문장이 나오면:

```text
We used a Wilcoxon signed-rank test on paired task scores.
```

이렇게 읽으면 된다.

```text
같은 gold task들에 대해 기존 방식과 새 방식의 점수 차이를 비교했다.
점수 차이가 정규분포라고 강하게 가정하지 않기 위해 Wilcoxon을 썼다.
```

그리고 다음을 확인하면 좋다.

```text
1. 정말 같은 task에서 old/new를 비교했나?
2. 각 task별 점수는 어떻게 계산했나?
3. 차이값이 0인 task는 어떻게 처리했나?
4. p-value만 말하지 않고 실제 점수 차이도 보여주나?
5. 17개 task가 작다는 한계를 인정하나?
```

> [!warning]
> Wilcoxon을 썼다고 해서 실험이 자동으로 완벽해지는 것은 아니다.
>
> gold task가 공정한지, selection bias가 없는지, 여러 비교 중 좋아 보이는 것만 고른 것은 아닌지도 같이 봐야 한다.

## 한 번에 정리

| 개념 | 뜻 | 기억할 점 |
| --- | --- | --- |
| paired test | 같은 대상에서 두 조건을 비교 | 같은 17개 문제를 old/new가 같이 풀면 paired |
| Wilcoxon signed-rank test | paired 데이터의 차이 방향과 순위를 보는 비모수 검정 | 새 방식이 일관되게 나은지 확인 |
| 비모수 검정 | 정규분포 가정을 덜 하는 검정 | 작은 표본이나 듬성듬성한 점수에 유용 |
| t-test | 평균 차이를 보는 모수 검정 | 차이값 정규분포 가정이 들어감 |

## 나중에 다시 볼 포인트

- 같은 gold task를 두 검색기가 같이 풀면 paired test다.
- paired test에서는 task별 old/new 차이를 본다.
- Wilcoxon signed-rank test는 차이값의 방향과 순위를 본다.
- 검색 평가 점수는 0~1 범위에 갇혀 있고 값이 듬성듬성할 수 있다.
- task 17개처럼 표본이 작으면 정규분포 가정을 강하게 두기 어렵다.
- Wilcoxon을 썼어도 실험 설계 자체가 공정한지는 따로 봐야 한다.
