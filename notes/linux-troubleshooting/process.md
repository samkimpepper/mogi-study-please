# 프로세스 확인

프로세스 확인은 "서버에서 어떤 프로그램이 어떤 PID로 떠 있는지"를 확인하는 작업이다.

장애 상황에서는 특정 PID가 CPU나 메모리를 많이 쓰는 경우가 많기 때문에, 그 PID가 어떤 애플리케이션인지 확인할 수 있어야 한다.

## ps

`ps`는 현재 떠 있는 프로세스를 확인하는 명령어다.

포트 상태를 직접 보는 도구라기보다는, 특정 PID가 어떤 프로세스인지 확인할 때 많이 쓴다.

```bash
ps -ef | grep java
ps -ef | grep node
ps -fp <PID>
```

예를 들어 어떤 포트가 특정 PID에 의해 잡혀 있다면, 그 PID의 정체를 확인할 때 쓴다.

```bash
ps -fp 12345
```

## top

`top`은 리눅스 터미널에서 보는 작업 관리자에 가깝다.

```bash
top
```

확인할 수 있는 것:

```text
- 전체 CPU 사용률
- 전체 메모리 사용량
- load average
- 프로세스별 CPU 사용률
- 프로세스별 메모리 사용률
- PID
- 실행 유저
- 실행 명령
```

장애 상황에서는 보통 이렇게 본다.

```text
서버가 느림
-> top 실행
-> CPU나 메모리가 터졌는지 확인
-> 어떤 PID가 많이 쓰는지 확인
-> ps -fp <PID>로 어떤 애플리케이션인지 확인
-> 로그나 설정을 이어서 확인
```

## kill

`kill`은 프로세스를 종료하는 명령어다.

개발할 때 "8080 포트 이미 사용 중" 같은 에러가 나면 보통 이 흐름으로 확인한다.

```bash
lsof -i :8080
ps -fp <PID>
kill <PID>
```

강제 종료가 필요할 때는 `kill -9`를 쓰기도 하지만, 가능하면 일반 `kill`부터 시도하는 편이 낫다.

```bash
kill <PID>
kill -9 <PID>
```

`kill -9`는 프로세스에게 정리할 시간을 주지 않고 강제로 종료하는 방식이라, 실무에서는 신중하게 써야 한다.
