# C 배열 길이와 Java 배열의 차이

> [!question]
> 내가 물어본 질문:
>
> - C언어에는 배열 길이를 반환하는 함수가 없는가?
> - 미리 확보한 메모리의 전체 크기밖에 못 가져온다는 말은 무슨 뜻인가?
> - Java 배열이나 `ArrayList`와는 무엇이 다른가?

> [!summary]
> 결론:
>
> - C 배열은 자기 길이를 런타임 정보로 들고 있지 않다.
> - 함수 밖의 실제 배열에 대해서는 `sizeof(arr) / sizeof(arr[0])`로 확보된 전체 칸 수를 계산할 수 있다.
> - 하지만 함수에 배열을 넘기면 대부분 포인터로 바뀌므로, 함수 안에서는 배열 길이를 알 수 없다.
> - Java 배열은 객체이고 `length` 필드를 가지므로 길이를 알 수 있다.
> - Java `ArrayList`는 실제 사용 개수인 `size`와 내부 배열의 여유 공간인 `capacity`를 구분한다.

관련 노트: [[C 배열이 길이 메타데이터를 들고 있지 않는 이유]]

## 먼저 잡을 한 줄 정의

> [!info]
> C 배열은 "연속된 메모리 공간"에 가깝고, Java 배열은 "길이 정보를 가진 객체"에 가깝다.

## C 배열에는 왜 길이 함수가 없나

C 배열은 자기 자신 안에 길이 정보를 저장하지 않는다.

```c
int arr[5] = {1, 2, 3, 4, 5};
```

이 코드는 `int` 5개가 들어갈 연속된 메모리 공간을 만든다.

함수 밖에서 실제 배열에 대해 `sizeof`를 쓰면 배열 전체 바이트 크기를 알 수 있다.

```c
size_t len = sizeof(arr) / sizeof(arr[0]);
```

```text
sizeof(arr)    = int 5개 전체 크기
sizeof(arr[0]) = int 1개 크기
```

그래서 이 경우에는 길이처럼 보이는 값을 계산할 수 있다.

> [!warning]
> 이 값은 배열이 자기 길이를 알려준 것이 아니라, 컴파일러가 현재 스코프에서 `arr`가 실제 배열이라는 것을 알고 있어서 전체 바이트 크기를 계산해준 것이다.

## 함수에 넘기면 포인터가 된다

함수 파라미터에서 배열처럼 써도 실제로는 포인터로 취급된다.

```c
void print_len(int arr[]) {
    printf("%zu\n", sizeof(arr) / sizeof(arr[0]));
}
```

위 코드는 사실상 다음과 비슷하다.

```c
void print_len(int *arr) {
    printf("%zu\n", sizeof(arr) / sizeof(arr[0]));
}
```

함수 안에서 `sizeof(arr)`는 배열 전체 크기가 아니라 포인터 크기다.

```text
함수 밖:
arr = 실제 배열
sizeof(arr) = 배열 전체 크기

함수 안:
arr = 포인터
sizeof(arr) = 포인터 크기
```

그래서 C에서는 배열을 함수에 넘길 때 보통 길이를 함께 넘긴다.

```c
void print_arr(int *arr, size_t len) {
    for (size_t i = 0; i < len; i++) {
        printf("%d\n", arr[i]);
    }
}

int main(void) {
    int arr[5] = {1, 2, 3, 4, 5};
    print_arr(arr, 5);
}
```

## 미리 확보한 메모리의 전체 크기밖에 못 가져온다는 말

이 말은 보통 다음 뜻에 가깝다.

> [!info]
> C에서 `sizeof(arr)`로 알 수 있는 것은 "배열에 확보된 전체 바이트 수"이지, "현재 의미 있게 들어 있는 원소 개수"가 아니다.

예를 들어 다음 배열은 10칸을 확보한다.

```c
int arr[10];

arr[0] = 1;
arr[1] = 2;
arr[2] = 3;
```

실제로 의미 있게 넣은 값은 3개지만, 배열 자체는 10칸이다.

```text
[1][2][3][?][?][?][?][?][?][?]
```

따라서 함수 밖에서 다음 계산을 하면 10이 나온다.

```c
sizeof(arr) / sizeof(arr[0])
```

C는 다음 정보는 알 수 없다.

```text
지금 실제로 몇 개를 사용 중인가?
몇 번째까지 유효한 데이터인가?
논리적인 길이는 몇 개인가?
```

즉 `sizeof`로 알 수 있는 것은 `capacity`에 가깝고, 실제 사용 개수인 `size`는 개발자가 따로 관리해야 한다.

## 문자열은 왜 `strlen`이 있나

C 문자열은 일반 배열과 다르게 끝 표시를 둔다.

```c
char s[] = "hello";
```

실제 메모리는 다음과 비슷하다.

```text
'h' 'e' 'l' 'l' 'o' '\0'
```

`strlen(s)`는 배열의 전체 크기를 아는 것이 아니다. 앞에서부터 읽다가 널 문자 `'\0'`를 만날 때까지 센다.

```c
strlen(s); // 5
```

> [!warning]
> `strlen`은 문자열의 논리적 길이를 센다.
> 일반 `int arr[]` 같은 배열에는 끝 표시가 없으므로 같은 방식으로 길이를 알 수 없다.

## Java 배열과의 차이

Java 배열은 객체다.

```java
int[] arr = {1, 2, 3, 4, 5};

System.out.println(arr.length); // 5
```

Java 배열 객체는 길이 정보를 가지고 있으므로 `arr.length`로 길이를 알 수 있다.

```text
Java int[] 객체
├─ length: 5
└─ 실제 원소들
```

그래서 Java에서는 메서드에 배열을 넘겨도 길이를 알 수 있다.

```java
void printArr(int[] arr) {
    System.out.println(arr.length);
}
```

C와 다르게 배열이 단순히 첫 번째 원소 주소만 전달되는 것이 아니라, 길이 정보를 가진 배열 객체에 대한 참조가 전달된다.

## Java ArrayList와 비교하기

`ArrayList`는 내부적으로 배열을 사용하지만, 실제 사용 개수인 `size`를 따로 관리한다.

```java
ArrayList<Integer> list = new ArrayList<>(10);
list.add(1);
list.add(2);
list.add(3);
```

이 경우 감각은 다음과 같다.

```text
capacity = 10
size = 3
```

`list.size()`는 실제로 들어 있는 원소 개수를 반환한다.

```java
System.out.println(list.size()); // 3
```

C 배열에서 `sizeof(arr) / sizeof(arr[0])`로 구하는 값은 `ArrayList`의 `size`보다는 `capacity`에 더 가깝다.

## 비교해서 이해하기

| 구분 | C 배열 | Java 배열 | Java ArrayList |
| --- | --- | --- | --- |
| 정체 | 연속된 메모리 공간 | 길이 정보를 가진 객체 | 내부 배열과 size를 관리하는 컬렉션 |
| 길이 정보 | 배열 자체가 런타임에 들고 있지 않음 | `length`로 알 수 있음 | `size()`로 실제 사용 개수 알 수 있음 |
| 함수/메서드 전달 | 대부분 포인터로 decay | 배열 객체 참조 전달 | 객체 참조 전달 |
| 전체 확보 크기 | 함수 밖 실제 배열이면 `sizeof`로 계산 가능 | `length`가 전체 길이 | 내부 capacity는 보통 직접 보지 않음 |
| 실제 사용 개수 | 개발자가 따로 관리 | 배열 길이와 같다고 보는 경우가 많음 | `size`로 관리 |

## 백엔드 관점으로 보면

> [!example]
> C 배열은 길이 없는 raw buffer에 가깝다.
>
> ```text
> buffer pointer + length
> ```
>
> 를 함께 넘겨야 안전하게 처리할 수 있다.

Java의 `ArrayList`는 내부적으로 다음 정보를 같이 들고 있는 구조에 가깝다.

```text
ArrayList
├─ elementData: 실제 배열
└─ size: 현재 사용 중인 원소 개수
```

그래서 Java 개발자는 `list.size()`를 호출하면 되지만, C에서는 이런 메타데이터를 직접 같이 들고 다녀야 한다.

## 면접에서 말하면

> [!tip]
> C 배열은 자기 길이를 런타임 정보로 저장하지 않기 때문에 표준적인 배열 길이 반환 함수가 없습니다. 함수 밖에서 실제 배열에 대해 `sizeof(arr) / sizeof(arr[0])`로 확보된 전체 원소 수를 계산할 수는 있지만, 함수에 배열을 넘기면 포인터로 decay되어 길이를 알 수 없습니다. 그래서 C에서는 배열 포인터와 길이를 함께 넘기는 패턴을 많이 씁니다. 반면 Java 배열은 객체이고 `length` 정보를 가지며, `ArrayList`는 실제 사용 개수인 `size`를 별도로 관리합니다.

## 내가 다시 헷갈릴 것 같은 부분

- C의 `sizeof(arr)`는 "현재 들어 있는 데이터 개수"가 아니라 "확보된 배열 전체 바이트 크기"를 알려준다.
- 함수에 배열을 넘기면 배열 전체가 넘어가는 것이 아니라 첫 번째 원소를 가리키는 포인터처럼 취급된다.
- C 문자열의 `strlen`은 배열 길이를 아는 것이 아니라 `'\0'`까지 세는 것이다.
- Java 배열은 길이 정보를 가진 객체라서 `length`를 알 수 있다.
- C 배열의 `sizeof` 계산 결과는 `ArrayList.size()`보다 `capacity`에 더 가까운 감각이다.
