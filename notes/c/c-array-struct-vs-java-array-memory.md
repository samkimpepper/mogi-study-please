# C 배열/구조체와 Java 배열의 저장 위치 차이

> 내가 물어본 질문:
>
> - C 구조체와 Java 배열은 둘 다 힙에 저장되는가?
> - C 배열도 힙에 저장될 수 있는가?
> - C만 선언 방식에 따라 저장 위치가 달라지는가?
> - Java 배열도 선언 방식에 따라 저장 위치가 달라지는 연속 메모리 공간인가?

## 한 줄 요약

C 배열과 C 구조체는 **어떻게 만들었는지**에 따라 스택, 힙, 정적 저장 영역에 놓일 수 있다. Java 배열은 C 배열처럼 선언 위치에 따라 본체 위치가 바뀌는 raw 메모리 공간이라기보다, `new`로 만들어지는 **힙 객체**로 보는 게 맞다.

## C는 선언 방식에 따라 위치가 달라진다

C 배열이나 구조체는 자료형 자체가 힙 전용이나 스택 전용으로 정해져 있지 않다.

지역 변수로 만들면 보통 스택에 놓인다.

```c
void test(void) {
    int arr[5];
}
```

```text
스택:
[int][int][int][int][int]
```

구조체도 똑같다.

```c
struct User {
    int id;
    char name[20];
    int age;
};

void test(void) {
    struct User user;
}
```

```text
스택:
[id][name 20칸][age]
```

`malloc`으로 만들면 힙에 놓인다.

```c
int *arr = malloc(sizeof(int) * 5);
```

```text
스택:
arr 포인터

힙:
[int][int][int][int][int]
```

구조체도 `malloc`으로 잡으면 힙이다.

```c
struct User *user = malloc(sizeof(struct User));
```

```text
스택:
user 포인터

힙:
[id][name 20칸][age]
```

전역 변수나 `static`으로 만들면 스택도 힙도 아니라 정적 저장 영역에 놓인다.

```c
int global_arr[5];

static struct User cached_user;
```

## Java 배열은 힙 객체다

Java에서 배열은 객체다.

```java
void test() {
    int[] arr = new int[5];
}
```

여기서 `arr`는 배열 본체가 아니라 배열 객체를 가리키는 참조 변수다.

```text
스택:
arr 참조 변수

힙:
int[] 배열 객체
[객체 헤더][length][0][0][0][0][0]
```

중요한 차이는 이것이다.

```java
int[] arr;
```

이 코드는 배열을 만든 것이 아니다. 배열을 가리킬 참조 변수만 선언한 것이다.

진짜 배열 객체는 `new`에서 만들어진다.

```java
arr = new int[5];
```

그래서 Java 배열은 C 배열처럼 "지역 변수로 선언했으니 배열 본체가 스택에 있다"라고 보지 않는다. 지역 변수 `arr`는 스택 프레임에 있을 수 있지만, `new int[5]`로 만든 배열 객체는 힙에 있다.

## C 배열과 Java 배열은 둘 다 연속 메모리와 관련은 있다

C 배열은 메타데이터 없는 연속 메모리 공간에 가깝다.

```text
C int arr[5]:
[값][값][값][값][값]
```

Java 배열도 원소 접근을 빠르게 하기 위해 내부 원소들은 연속 배치에 가깝게 다룬다고 보면 된다.

```text
Java int[]:
[객체 헤더][length][값][값][값][값][값]
```

하지만 둘의 핵심 차이는 "연속 메모리냐 아니냐"가 아니다.

```text
C 배열
-> 메타데이터 없는 연속 공간
-> 저장 위치는 선언/할당 방식에 따라 달라짐

Java 배열
-> length를 가진 배열 객체
-> 배열 객체는 보통 힙에 생성됨
```

## 구조체와 Java 배열은 직접 대응되는 관계가 아니다

C 구조체는 여러 필드를 묶어서 새 타입을 만드는 도구다.

```c
struct User {
    int id;
    char name[20];
    int age;
};
```

Java 배열은 같은 타입 원소를 순서대로 담는 객체다.

```java
int[] scores = new int[5];
```

그래서 더 자연스러운 비교는 다음에 가깝다.

```text
C 배열          <-> Java 배열
C 구조체        <-> Java class/object
C 구조체 + 배열 <-> Java 배열 객체 비슷한 수동 구현
```

C에서 Java 배열처럼 길이와 데이터를 같이 들고 다니고 싶으면 구조체로 직접 만들 수 있다.

```c
struct IntArray {
    size_t length;
    int data[5];
};
```

하지만 이 `length`는 Java 배열의 `length`처럼 언어/JVM이 보장하는 정보가 아니다. 개발자가 직접 넣고 직접 맞춰야 하는 필드다.

```c
struct IntArray arr;
arr.length = 999;  // 가능하지만 data는 5칸뿐이다.
```

Java 배열의 `length`는 직접 바꿀 수 없다.

```java
int[] arr = new int[5];
// arr.length = 999; // 불가능
```

## 비교 정리

| 구분 | C 배열/구조체 | Java 배열 |
| --- | --- | --- |
| 저장 위치 | 선언/할당 방식에 따라 달라짐 | 배열 객체는 보통 힙 |
| 지역 변수로 만들면 | 본체가 보통 스택 | 참조 변수만 지역 변수, 배열 본체는 힙 |
| 힙에 만들려면 | `malloc` 사용 | `new` 사용 |
| 전역/static | 정적 저장 영역 | 일반적인 배열 객체 생성과는 다른 관점 |
| 길이 정보 | C 배열은 없음. 필요하면 직접 관리 | `length`를 가짐 |
| 정체 | raw 메모리 공간 또는 값 묶음 | JVM이 관리하는 객체 |

## 결론

"선언 방식에 따라 저장 위치가 달라지는 연속 메모리 공간"이라는 감각은 C에 더 가깝다.

Java 배열은 내부 원소가 연속 배치에 가깝게 저장될 수 있지만, C 배열처럼 선언 위치에 따라 본체가 스택/힙으로 갈리는 raw 메모리 공간은 아니다. Java 배열은 `new`로 만들어지는 길이 정보를 가진 힙 객체라고 보는 게 맞다.

## 한 줄 회고

- 헷갈렸던 점: C 배열/구조체는 선언 방식에 따라 저장 위치가 달라지지만, Java 배열은 지역 변수 선언과 배열 객체 생성을 나눠 봐야 하고 배열 본체는 `new`로 만들어지는 힙 객체라는 점.
