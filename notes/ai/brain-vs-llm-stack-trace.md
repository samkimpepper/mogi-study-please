# 사람 뇌와 LLM은 왜 스택트레이스를 찍기 어려운가

> [!summary]
> 사람 뇌와 LLM은 둘 다 사람이 설계한 함수 단위로 동작하지 않는 분산 시스템이라, 프로그램처럼 깔끔한 스택트레이스를 찍기 어렵다.
>
> 다만 LLM은 외부에서 가중치와 활성화를 관측할 수 있지만 해석이 어려운 문제에 가깝고, 뇌는 자기 자신을 관측하는 통로 자체도 제한되어 있다는 차이가 있다.

## 처음 헷갈렸던 질문

> [!question]
> LLM은 자기 사고과정을 스택트레이스처럼 못 찍는다고 했는데, 사람 뇌는 가능한 걸까?
>
> 사람이 자기 생각의 진짜 실행 로그를 볼 수 있을까?

결론부터 말하면 사람 뇌도 어렵다.

다만 LLM과 뇌는 "못 찍는 이유"가 완전히 같지는 않다.

```text
공통점:
깔끔한 함수 경계가 없다.

차이점:
LLM은 외부 관측은 가능하지만 해석이 어렵다.
뇌는 외부 관측도 어렵고, 자기 관측 통로도 제한되어 있다.
```

## 공통점: 함수 경계가 없다

프로그램에서 stack trace가 찍히려면 이름 있고 경계가 명확한 함수들이 있어야 한다.

```text
handleRequest()
-> calculateTax()
-> applyDiscount()
-> saveOrder()
```

이런 구조라면 어디서 예외가 났는지 추적할 수 있다.

하지만 LLM과 뇌는 이런 식으로 사람이 이름 붙인 함수 단위로 나뉘어 있지 않다.

LLM은 수많은 파라미터와 활성화가 동시에 얽혀서 다음 토큰 분포를 만든다.

뇌도 뉴런과 시냅스가 거대한 네트워크를 이루고, 하나의 생각이 여러 영역에 분산되어 처리된다.

> [!info]
> LLM과 뇌는 모두 사람이 의미 단위로 설계한 함수형 시스템이라기보다, 학습이나 진화로 만들어진 분산 시스템에 가깝다.

그래서 "기억 담당 함수", "결정 담당 함수", "추론 담당 함수"처럼 깔끔한 호출 경계를 기대하기 어렵다.

이 점은 LLM의 superposition과도 비슷하다.

하나의 뉴런, 회로, 활성화 패턴이 하나의 명확한 의미만 담당하기보다 여러 기능이 겹쳐 나타날 수 있다.

## LLM은 관측은 가능하지만 해독이 어렵다

LLM은 원리상 내부 숫자를 볼 수 있다.

```text
가중치
활성화 값
attention pattern
logit
토큰 확률
```

이 값들은 모델 파일과 실행 과정에서 관측할 수 있다.

문제는 숫자를 볼 수 있다는 것과 그 의미를 이해한다는 것이 다르다는 점이다.

```text
관측 가능:
값을 읽을 수 있음

해석 어려움:
그 값이 왜 특정 답변으로 이어졌는지 사람 의미 단위로 설명하기 어려움
```

> [!warning]
> LLM은 "안을 못 봐서"만 문제가 되는 것이 아니다.
>
> 안을 볼 수 있어도 그 숫자들이 사람이 이해하는 이유, 함수, 의도 단위로 정리되어 있지 않기 때문에 해석이 어렵다.

이것이 mechanistic interpretability가 어려운 이유 중 하나다.

## 뇌는 관측 통로도 제한되어 있다

사람 뇌는 LLM보다 한 겹 더 어렵다.

LLM은 외부 연구자가 모델 내부 숫자를 읽을 수라도 있다. 하지만 사람 뇌는 외부에서 세밀하게 관측하기가 훨씬 어렵다.

```text
fMRI:
공간/시간 해상도가 거칠다.

EEG:
시간 해상도는 좋지만 공간 해석이 어렵다.

개별 뉴런 수준 관측:
일반적인 인간 사고 전체에 대해 쉽게 할 수 없다.
```

또한 자기 자신을 관측하는 통로도 제한되어 있다.

우리가 의식적으로 아는 것은 뇌 전체 처리 과정의 극히 일부다.

> [!info]
> 의식은 전체 내부 로그를 보는 root 권한이 아니라, 일부 요약 대시보드를 보는 사용자에 가깝다.

백엔드 비유로 보면 다음과 같다.

```text
실제 내부 처리:
DB 트랜잭션
커넥션 풀
GC
락 경합
스레드 스케줄링

의식이 보는 것:
대시보드에 올라온 최종 상태와 일부 요약
```

그래서 "왜 그렇게 결정했어?"라고 물으면, 뇌는 내부 실행 로그를 그대로 읽어서 답하는 것이 아니다.

의식에 올라온 결과와 단서를 바탕으로 그럴듯한 이유를 사후에 구성한다.

## 내성은 스택트레이스가 아니다

사람은 자기 마음을 들여다보면 이유를 알 수 있다고 느낀다.

하지만 내성으로 얻는 이유는 진짜 실행 로그라기보다 사후 내레이션에 가깝다.

```text
무의식적 처리
-> 결과가 의식에 올라옴
-> 의식이 이유를 설명함
```

이때 설명은 실제 원인과 어느 정도 맞을 수도 있지만, 완전히 정확한 실행 로그라고 보기는 어렵다.

> [!warning]
> "내가 이렇게 느꼈으니 이유도 정확히 안다"는 것은 위험할 수 있다.
>
> 느낀 결과와 실제 처리 메커니즘은 다른 층위다.

LLM도 비슷한 증상을 보인다.

```text
모델 내부 계산
-> 답변 생성
-> "왜 그렇게 답했는지" 텍스트 생성
```

이 설명은 유용할 수 있지만, 내부 계산의 정확한 stack trace는 아니다.

## LLM과 뇌의 비교

| 구분 | LLM | 인간 뇌 |
| --- | --- | --- |
| 함수 경계 | 없음, 분산/중첩 | 없음, 분산 처리 |
| 외부 관측 | 가중치와 활성화 관측 가능 | 매우 제한적 |
| 해석 난이도 | 관측은 되지만 해독이 어려움 | 관측도 어렵고 해독도 어려움 |
| 자기 내부 관측 | 내부 숫자를 직접 읽는 통로 없음 | 의식이 무의식 전체에 접근하지 못함 |
| 이유 설명 | 사후 생성된 텍스트 | 사후 생성된 내레이션 |

핵심 차이는 다음과 같다.

> [!summary]
> LLM은 "관측은 되는데 해독이 안 되는" 문제에 가깝다.
>
> 인간 뇌는 여기에 더해 "자기 관측 통로 자체가 제한된" 문제까지 있다.

## 정보 은닉은 버그가 아니라 기능일 수 있다

사람이 자기 내부 처리를 전부 의식하지 못하는 것은 비효율을 줄이기 위한 구조로 볼 수도 있다.

예를 들어 걸을 때마다 다음 로그가 의식에 전부 올라온다고 해보자.

```text
대퇴사두근 발화율 조정
전정기관 균형 보정
발목 각도 미세 조정
시각 입력 기반 경로 수정
```

이런 내부 로그를 전부 의식하면 오히려 행동이 느려지고 복잡해질 수 있다.

의식에는 추상화된 상태만 올라오는 편이 효율적이다.

```text
그냥 걷는다.
앞에 장애물이 있다.
오른쪽으로 피한다.
```

백엔드나 객체지향의 정보 은닉과 비슷하게 볼 수 있다.

```text
외부 인터페이스:
walk()

내부 구현:
수많은 근육 제어, 균형 보정, 감각 통합
```

> [!tip]
> 내부 구현을 모두 노출하지 않는 것은 버그가 아니라 기능일 수 있다.
>
> 뇌도 모든 처리 로그를 의식에 올리기보다, 필요한 요약만 올리는 방식으로 동작한다고 볼 수 있다.

## 전체 대화와 이어지는 결론

LLM과 사람 뇌는 메커니즘이 다르지만, 중요한 공통 패턴이 있다.

```text
실제로 동작하는 메커니즘
설명하는 메커니즘
```

이 둘은 같은 것이 아니다.

LLM이 "왜 그렇게 답했는지" 설명하는 것과 실제 내부 계산은 다르다.

사람이 "내가 왜 그렇게 선택했는지" 말하는 것과 실제 뇌의 전체 처리 과정도 다르다.

> [!summary]
> 설명하는 행위와 실제로 동작하는 메커니즘은 별개의 프로세스다.
>
> 이 원칙은 LLM에도 적용되고, 사람의 자기 설명에도 어느 정도 적용된다.

## 나중에 다시 볼 포인트

- stack trace는 이름 있고 경계가 명확한 함수 호출 구조가 있을 때 잘 찍힌다.
- LLM과 뇌는 모두 분산 시스템이라 깔끔한 함수 경계가 없다.
- LLM은 내부 숫자를 관측할 수 있지만, 사람이 이해하는 의미 단위로 해독하기 어렵다.
- 뇌는 외부 관측도 어렵고, 의식이 무의식의 전체 처리 과정에 접근하지 못한다.
- 사람의 자기 설명도 진짜 실행 로그라기보다 사후 내레이션일 수 있다.
- 설명과 실제 동작 메커니즘은 분리해서 봐야 한다.

## 복습 질문

- [ ] LLM과 뇌가 모두 분산 시스템이라는 말의 의미를 설명할 수 있다.
- [ ] LLM은 관측 가능하지만 해독이 어렵다는 말을 설명할 수 있다.
- [ ] 뇌의 자기 관측 통로가 제한되어 있다는 말을 백엔드 비유로 설명할 수 있다.
- [ ] 설명하는 행위와 실제 동작 메커니즘이 다르다는 결론을 LLM과 사람 뇌에 각각 적용할 수 있다.
