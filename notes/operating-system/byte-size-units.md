# 바이트 크기 단위 정리

> [!summary]
> 컴퓨터에서 데이터 크기를 말할 때 기본 단위는 byte다.
>
> `KB`, `MB`, `GB`는 byte가 커졌을 때 쓰는 단위이고, 메모리와 파일 크기에서는 1000 단위와 1024 단위가 섞여 나와 헷갈릴 수 있다.

## 처음 헷갈렸던 질문

> [!question]
> 64B, 4KB, 48KB, 2.5MB 같은 숫자가 계속 나오는데 각각 얼마나 큰 건가?
>
> KB는 1000바이트인가, 1024바이트인가?

## 기본 단위

가장 먼저 잡을 단위는 bit와 byte다.

| 단위 | 의미 | 기억할 점 |
| --- | --- | --- |
| bit | 0 또는 1 하나 | 가장 작은 정보 단위 |
| byte | 8 bit | 보통 파일, 메모리 크기의 기본 단위 |

```text
1 byte = 8 bit
```

예를 들어 Java의 `byte` 하나는 8bit 크기다.

## B, KB, MB, GB

일상적으로는 다음처럼 기억하면 된다.

| 단위 | 대략 크기 | 감각 |
| --- | --- | --- |
| `B` | 1 byte | 아주 작은 데이터 |
| `KB` | 약 1천 byte | 짧은 텍스트, 작은 설정 파일 |
| `MB` | 약 100만 byte | 이미지, 작은 문서, 라이브러리 파일 |
| `GB` | 약 10억 byte | 메모리, 큰 파일, DB 데이터 |
| `TB` | 약 1조 byte | 디스크, 대용량 스토리지 |

## 1000 단위와 1024 단위

여기서 헷갈리는 이유는 `KB`, `MB`, `GB`가 상황에 따라 두 방식으로 쓰이기 때문이다.

### 10진 단위

디스크나 네트워크 속도에서는 보통 1000 단위를 쓴다.

```text
1 KB = 1,000 B
1 MB = 1,000 KB = 1,000,000 B
1 GB = 1,000 MB = 1,000,000,000 B
```

### 2진 단위

메모리, 페이지, 캐시 같은 OS/CPU 문맥에서는 보통 1024 단위로 생각하는 경우가 많다.

```text
1 KiB = 1,024 B
1 MiB = 1,024 KiB = 1,048,576 B
1 GiB = 1,024 MiB = 1,073,741,824 B
```

정확한 표기는 1024 단위에는 `KiB`, `MiB`, `GiB`를 쓰는 것이 맞다. 하지만 실제 문서나 대화에서는 그냥 `KB`, `MB`, `GB`라고 쓰면서 1024 단위를 의미하는 경우도 많다.

> [!warning]
> OS/메모리 문맥에서 `4KB page`, `64KB`, `48KB cache`처럼 나오면 보통 1024 단위로 이해하면 된다.
>
> 하지만 디스크 제조사나 네트워크 속도에서는 1000 단위일 가능성이 높다.

## 자주 나오는 크기 감각

CPU 캐시와 OS 메모리 공부에서 자주 보는 숫자는 다음과 같다.

| 크기 | byte로 보면 | 주로 나오는 곳 |
| --- | --- | --- |
| `1B` | 1 byte | 문자 일부, 작은 정수 조각 |
| `4B` | 4 byte | `int`, `uint32_t`, IPv4 주소 |
| `8B` | 8 byte | `long`, pointer, PTE 하나 |
| `64B` | 64 byte | x86에서 흔한 캐시 라인 크기 |
| `128B` | 128 byte | M1에서 보고된 캐시 라인 크기 |
| `4KB` | 4,096 byte | x86에서 흔한 페이지 크기 |
| `16KB` | 16,384 byte | M1/macOS에서 보고된 페이지 크기 |
| `48KB` | 49,152 byte | 예시 L1d 캐시 크기 |
| `2.5MB` | 약 2.5 * 1024 * 1024 byte | 예시 L2 캐시 크기 |

## 왜 4KB는 4096B인가

OS/메모리 문맥의 `4KB`는 보통 1024 단위를 뜻한다.

```text
4KB = 4 * 1024B = 4096B
```

이 값은 페이지 크기나 캐시 set 계산에서 자주 나온다.

예를 들어 캐시 라인이 64B이고 set이 64개라면:

```text
64B * 64 = 4096B = 4KB
```

그래서 주소가 4096B 차이 나면 같은 캐시 set으로 돌아올 수 있다는 설명이 나왔다.

## 왜 64B 캐시 라인에 int가 16개 들어가는가

`int`나 `uint32_t`는 보통 4B다.

캐시 라인 하나가 64B라면:

```text
64B / 4B = 16
```

그래서 정수 16개가 캐시 라인 하나에 들어간다.

```text
cache line 1
data[0] data[1] data[2] ... data[15]
```

이 때문에 `data[0]`을 읽으려고 캐시 라인을 가져오면, `data[1]`부터 `data[15]` 근처도 같이 들어올 수 있다.

## 대문자 B와 소문자 b

이것도 자주 헷갈린다.

| 표기 | 의미 |
| --- | --- |
| `B` | byte |
| `b` | bit |

```text
1B = 8b
```

네트워크 속도에서 `Mbps`라고 하면 보통 megabit per second다. 파일 크기에서 `MB`라고 하면 megabyte다.

```text
100 Mbps = 100 megabit/s
100 Mbps = 12.5 MB/s 정도
```

왜냐하면 8bit가 1byte이기 때문이다.

## 백엔드 개발자가 기억할 정도

다 외울 필요는 없고, 다음 정도면 충분하다.

```text
1 byte = 8 bit
1KB는 메모리 문맥에서 보통 1024B로 생각한다.
4KB = 4096B
64B 캐시 라인에는 4B int가 16개 들어간다.
대문자 B는 byte, 소문자 b는 bit다.
네트워크 Mbps와 파일 MB/s는 8배 차이가 난다.
```

## 면접에서 말하면

> [!tip]
> 컴퓨터에서 메모리와 파일 크기의 기본 단위는 byte이고, 1 byte는 8 bit입니다. OS나 메모리 문맥에서는 KB, MB가 보통 1024 단위로 쓰이는 경우가 많아서 4KB 페이지는 4096 byte로 이해합니다. 반면 디스크나 네트워크 문맥에서는 1000 단위나 bit 단위가 섞여 나올 수 있어, 대문자 `B`는 byte, 소문자 `b`는 bit라는 점을 구분해야 합니다.
