# 🧬 진화는 탐색 알고리즘인가

> "진화가 뭔가 그리디 알고리즘처럼 느껴진다"는 직관에서 출발한 노트. 결론부터 말하면 절반은 맞고, 절반은 더 복잡하다.

---

## 📌 한 줄 요약

진화는 **그리디한 선택(selection)** 만 있는 순수 그리디가 아니다.

선택, 변이, 유전적 부동, 재조합을 개체군 단위로 돌리는 **확률적 병렬 탐색**에 가깝다.

> 개발자 언어로 줄이면: 진화 = exploit 중심의 선택 + explore를 만드는 mutation, drift, recombination.

---

## 1. CS에서도 진화에서 알고리즘을 만들었다

1970년대에 존 홀랜드(John Holland)는 "진화가 일종의 탐색 알고리즘 아닌가?"라는 관점에서 **유전 알고리즘(Genetic Algorithm)** 을 만들었다.

즉, 자연 진화에서 영감을 받아 실제 컴퓨터 과학의 최적화 기법이 생겼다.

유전 알고리즘은 대충 이런 식으로 돈다.

```text
후보 해 여러 개를 만든다
-> 각 후보의 적합도를 평가한다
-> 좋은 후보를 더 많이 남긴다
-> 변이와 재조합으로 새 후보를 만든다
-> 반복한다
```

이 구조가 생물 진화의 선택, 돌연변이, 유성생식과 닮아 있다.

---

## 2. "그리디 같다"가 맞는 부분 — 선택 단계

자연선택은 미래를 보지 않는다.

매 세대마다 그 순간 환경에서 더 잘 살아남고 더 잘 번식하는 변이가 더 많이 남는다.

이 점은 그리디와 닮았다.

```text
현재 상태에서 더 적합한 변이 선택
-> 다음 세대로 더 많이 전달
-> 다시 현재 상태에서 더 적합한 변이 선택
```

알고리즘으로 보면 **언덕 오르기(hill climbing)** 에 가깝다.

적합도 지형(fitness landscape) 위에서, 지금 위치보다 조금 더 높은 곳으로 이동하는 지역 탐색이다.

```text
낮은 곳
-> 조금 더 높은 곳
-> 더 높은 곳
-> 주변에서 제일 높은 곳
```

이때 "지난번 불가능 산" 같은 비유는 바로 fitness landscape로 볼 수 있다.

---

## 3. 그리디의 약점도 그대로 있다 — 지역 최적

그리디와 힐클라이밍은 **지역 최적(local optimum)** 에 갇힐 수 있다.

더 높은 봉우리가 멀리 있어도, 거기로 가려면 일단 계곡으로 내려가야 할 수 있다.

하지만 그리디는 당장 손해 보는 이동을 싫어한다.

```text
현재 봉우리
-> 주변은 전부 내려가는 길
-> 더 높은 봉우리는 멀리 있음
-> 내려가야 갈 수 있음
-> 그리디는 못 감
```

진화도 비슷하게 갇힐 수 있다.

대표적인 예시가 사람의 눈이다.

사람 눈의 망막은 빛을 받는 세포와 신경 배선 방향이 비효율적으로 얽혀 있다. 신경다발이 망막 앞을 지나가면서 맹점(blind spot)도 생긴다.

처음부터 다시 설계하면 더 깔끔하게 만들 수 있을 것 같지만, 진화는 처음부터 새로 짜지 않는다. 이미 돌아가는 구조를 조금씩 수정한다.

중간 단계가 더 안 보이는 눈이라면 선택에서 살아남기 어렵다.

> 레거시 코드를 한 번에 갈아엎으면 좋아질 것 같지만, 중간 배포 버전들이 전부 장애라서 못 가는 상황과 비슷하다.

반대로 문어 눈은 따로 진화해서 사람 눈과 다른 배선 구조를 갖고 있고, 같은 결함이 없다.

---

## 4. 그래도 순수 그리디보다는 똑똑하다

진화는 선택만으로 움직이지 않는다.

국소 최적에 갇히는 문제를 완전히 없애지는 못하지만, 빠져나갈 수 있는 장치들이 있다.

| 진화의 요소 | 알고리즘 관점 | 역할 |
| --- | --- | --- |
| 개체군 | 병렬 탐색, beam search | 여러 후보를 동시에 굴린다 |
| 돌연변이 | random perturbation | 새로운 방향을 랜덤하게 시도한다 |
| 유전적 부동 | simulated annealing 비슷한 효과 | 약간 손해인 변이도 우연히 퍼질 수 있다 |
| 재조합 | crossover | 서로 다른 좋은 부분을 섞는다 |

---

## 5. 병렬 탐색 — 후보 하나만 보는 게 아니다

순수 그리디는 보통 현재 해 하나만 들고 전진한다.

하지만 진화는 개체군 전체를 굴린다.

```text
개체 A: 봉우리 1 탐색
개체 B: 봉우리 2 탐색
개체 C: 계곡 근처 탐색
개체 D: 다른 방향 탐색
```

이건 beam search나 population-based search에 가깝다.

모든 후보가 한 곳에 동시에 갇히는 것을 줄이고, 여러 방향을 동시에 시험할 수 있다.

---

## 6. 유전적 부동 — 가끔 손해도 퍼진다 🎲

자연선택만 보면 항상 더 좋은 변이만 남을 것 같지만, 실제 진화에는 우연이 있다.

특히 작은 집단에서는 약간 손해인 변이도 우연히 퍼질 수 있다. 이것을 **유전적 부동(genetic drift)** 이라고 한다.

알고리즘 감각으로 보면, 이건 가끔 점수가 낮아지는 이동도 허용하는 것과 비슷하다.

그래서 시뮬레이티드 어닐링(simulated annealing)과 닮은 면이 있다.

```text
당장은 조금 손해
-> 계곡을 건넘
-> 더 높은 봉우리로 갈 가능성 생김
```

물론 진화가 일부러 전략적으로 손해를 감수하는 것은 아니다.

하지만 결과적으로는 지역 최적에서 빠져나오는 통로가 될 수 있다.

---

## 7. 재조합 — 좋은 조각 섞기

유성생식은 서로 다른 개체의 유전자를 섞는다.

알고리즘으로 보면 crossover다.

```text
후보 A의 좋은 부분
+ 후보 B의 좋은 부분
-> 후보 C
```

단순 힐클라이밍은 한 후보를 조금씩 바꾸는 쪽에 가깝다.

반면 유전 알고리즘은 여러 후보의 좋은 building block을 섞어서 새로운 후보를 만든다.

이게 유전 알고리즘이 단순 힐클라이밍보다 풍부한 탐색을 할 수 있는 이유다.

---

## 8. exploit과 explore로 보면 딱 맞다

그리디는 거의 순수 exploit이다.

지금 좋아 보이는 것을 계속 활용한다.

진화는 여기에 explore가 섞인다.

| 구분 | 의미 | 진화에서 대응 |
| --- | --- | --- |
| exploit | 현재 잘 되는 방향을 더 밀기 | 선택(selection) |
| explore | 새로운 가능성 탐색 | 돌연변이, 유전적 부동, 재조합 |

그래서 진화는 **탐험-활용 트레이드오프**를 가진 확률적 병렬 탐색으로 볼 수 있다.

---

## 9. 알고리즘과 끝내 다른 점 — 고정 목적 함수가 없다

일반적인 최적화 알고리즘은 목적 함수가 정해져 있다.

```text
minimize cost
maximize score
maximize fitness
```

하지만 자연 진화에는 고정된 최종 목표가 없다.

적합도는 "그 순간 환경에서 살아남고 번식하는 정도"에 가깝다.

문제는 환경이 계속 바뀐다는 점이다.

- 기후가 바뀐다.
- 먹이가 바뀐다.
- 천적이 바뀐다.
- 경쟁자도 같이 진화한다.

즉, 목적 함수 자체가 움직인다.

붉은 여왕 가설(Red Queen hypothesis)은 이 느낌을 잘 보여준다.

> 가만히 있으려면 계속 달려야 한다.

한 종이 좋아지면, 그 종의 천적이나 경쟁자도 같이 변한다.

그래서 진화는 정해진 정답으로 수렴하는 알고리즘이라기보다, 움직이는 과녁 위에서 계속 적응하는 멀티에이전트 최적화에 가깝다.

---

## 10. 백엔드 개발자 관점으로 다시 말하면

진화는 레거시 시스템 개선과 은근히 닮았다.

처음부터 새로 만들면 더 나은 구조가 있을 수 있다.

하지만 현실에서는 이미 운영 중인 시스템이 있고, 중간 배포 버전이 계속 살아남아야 한다.

```text
현재 시스템
-> 작은 변경
-> 운영에서 살아남음
-> 다시 작은 변경
-> 운영에서 살아남음
```

당장 장애를 내는 중간 단계는 선택되지 못한다.

그래서 더 좋은 최종 구조가 있어도, 거기까지 가는 경로가 모두 나쁘면 도달하기 어렵다.

이게 지역 최적에 갇힌 진화와 비슷하다.

---

## 11. 핵심 비유를 더 정확히 잡기 🏗️

진화는 처음부터 완벽하게 설계하는 global optimizer가 아니다.

오히려 **운영 중인 레거시 시스템에 작은 패치를 계속 배포하는 과정**에 가깝다.

```text
현재 동작하는 생물
-> 작은 변이 발생
-> 현재 환경에서 번식에 유리한 변이가 더 많이 남음
-> 반복
```

미래의 완벽한 설계도를 보고 움직이지 않기 때문에, 당장 개선되는 방향으로 가다가 국소최적에 머물 수 있다.

백엔드로 치면 "언젠가 완벽한 아키텍처가 된다"는 장기 설계도보다, **지금 프로덕션에서 살아남는 변경**이 먼저 선택되는 구조다.

---

## 12. 그리디 예시로 보는 국소최적

그리디 알고리즘은 각 단계에서 지금 가장 좋아 보이는 선택을 한다.

예를 들어 동전이 `[4, 3, 1]`이고 6원을 만들어야 한다고 하자.

```text
그리디: 4 + 1 + 1 = 동전 3개
최적해: 3 + 3     = 동전 2개
```

현재 단계에서 가장 큰 `4`를 고르는 것은 좋아 보인다.

하지만 전체 결과는 최적이 아니다.

다만 진화와 더 정확히 비슷한 알고리즘은 전형적인 그리디보다 **힐 클라이밍**이다.

```python
current = initial_solution

while better_neighbor_exists(current):
    current = best_neighbor(current)

return current
```

힐 클라이밍은 현재 상태 주변의 더 좋은 상태로 계속 이동한다.

문제는 주변에 더 좋은 상태가 없으면 멈춘다는 것이다.

---

## 13. 국소최적을 조금 엄밀하게 쓰면

상태 `x`의 작은 변형들을 `N(x)`, 성능을 `f(x)`라고 하자.

```text
모든 y ∈ N(x)에 대해 f(x) ≥ f(y)
```

이면 `x`는 국소최적이다.

```text
적합도
  ^
  |                       G: 전역최적
  |                      /\
  |       L             /  \
  |      / \___________/    \
  |     /                    \
  +------------------------------> 유전자·형질 상태
         ↑
       국소최적
```

`L`에서는 어느 방향으로 한 걸음 움직여도 성능이 떨어진다.

하지만 멀리 떨어진 `G`는 더 좋다.

문제는 `L -> G`로 가려면 중간의 낮은 구간, 즉 **적합도 계곡(fitness valley)** 을 건너야 한다는 점이다.

실제 진화 연구에서도 유전자형이나 형질에 여러 적응 봉우리가 존재할 수 있고, 환경 변화가 이 계곡을 낮추거나 우회로를 만들 수 있음이 관찰된다.

---

## 14. 왜 진화는 중간 단계를 못 건너뛰나

어떤 생물이 현재 잘 살아가고 있다고 하자.

```text
현재 기관 A: 적합도 100
조금 변경한 A1: 적합도 95
조금 변경한 A2: 적합도 92

완전히 재설계된 기관 B: 적합도 150
```

B가 훨씬 좋더라도 A에서 B로 즉시 점프할 수는 없다.

유전적 변화는 보통 기존 구조를 수정해서 나타난다.

중간 단계인 A1, A2가 생존이나 번식에 불리하면 자연선택에 의해 제거될 가능성이 높다.

자연선택은 이렇게 판단하지 않는다.

```text
"지금은 조금 손해지만 100만 년 후에는 훌륭한 구조가 됩니다."
```

현재 세대에서 작동해야 한다.

미래를 위한 장기 리팩터링 계획은 없다.

## 15. 백엔드 마이그레이션으로 보면

현재 시스템이 다음과 같다고 하자.

```text
거대한 모놀리스
+ 여러 서비스가 공유하는 DB
+ 모든 테이블이 오래된 정수 ID 사용
```

이상적인 구조는 다음과 같을 수 있다.

```text
명확한 도메인 경계
+ 독립 DB
+ 이벤트 기반 통신
+ 일관된 새로운 ID 체계
```

하지만 한 번에 옮기려고 하면 중간 상태가 복잡해진다.

```text
dual write
데이터 백필
구·신 API 동시 지원
이벤트 유실 가능성
정합성 문제
장애 가능성 증가
```

최종 구조는 더 좋아도 **중간 마이그레이션 상태가 더 나쁠 수 있다.**

서비스는 중단할 수 없으므로 현재의 "못생겼지만 안정적인 구조"가 유지될 수 있다.

```text
진화의 적합도 =
성능 + 안정성 + 에너지 비용 + 발달 가능성 + 번식 성공

백엔드의 적합도 =
성능 + 안정성 + 운영 비용 + 마이그레이션 위험 + 팀 역량
```

현재 아키텍처가 처리량만 보면 비효율적이어도, 마이그레이션 위험까지 포함하면 국소최적일 수 있다.

## 16. 생물 기관의 비효율 사례

### 16.1 되돌이후두신경

되돌이후두신경은 뇌에서 후두로 바로 연결되지 않는다.

가슴 방향으로 내려가 혈관 아래를 감싼 뒤 다시 후두로 올라온다.

목이 긴 기린에서는 이 우회 경로가 특히 길어진다.

기존 척추동물의 신경과 혈관 배치를 유지한 상태에서 목과 신체 구조가 변화한 결과로 설명된다.

처음부터 기린을 설계한다면 이렇게 만들 수 있을 것 같다.

```text
뇌 ----------> 후두
```

하지만 진화가 실제로 수정한 대상은 기존 배선이다.

```text
기존 신경 배선
-> 목이 조금 길어짐
-> 기존 배선도 조금 길어짐
-> 반복
```

전체 배선을 뜯어고치는 중간 단계는 정상적인 발생과 생존을 망가뜨릴 수 있다.

### 16.2 척추동물의 망막

척추동물의 망막은 광수용체가 빛의 반대쪽을 향하는 역전 구조다.

반면 두족류의 광수용체는 빛을 향하는 구조를 가진다.

동일한 시각 기능도 계통별 발달 역사에 따라 서로 다른 구조로 구현될 수 있다는 사례다.

이것은 같은 기능을 구현하는 두 레거시 코드베이스가 완전히 다른 내부 구조를 갖는 것과 비슷하다.

## 17. "비효율 = 국소최적 증명"은 너무 강하다 ⚖️

생물 기관이 비효율적으로 보인다는 사실만으로 국소최적이 수학적으로 증명되는 것은 아니다.

그 사실만으로 알 수 있는 것은 이 정도다.

```text
이 기관이 우리가 선택한 기준에서 전역최적처럼 보이지 않는다
```

정말 국소최적임을 증명하려면 다음을 확인해야 한다.

```text
1. 무엇을 적합도라고 할 것인가?
2. 어떤 변화를 '이웃 상태'라고 할 것인가?
3. 가능한 모든 작은 변화가 실제로 적합도를 낮추는가?
4. 멀리 떨어진 곳에 더 높은 적합도의 구조가 있는가?
5. 그곳으로 가는 경로가 낮은 적합도를 거쳐야 하는가?
```

복잡한 생물 기관에서 이 모든 것을 측정하는 것은 거의 불가능하다.

게다가 비효율은 여러 이유로 생길 수 있다.

- 한 기능의 최적화가 다른 기능을 해치는 트레이드오프
- 발생 과정상 만들 수 없는 구조라는 제약조건
- 선택압이 약해서 사라지지 않은 작은 결함
- 과거 환경에서는 유리했지만 현재 환경이 바뀐 경우
- 유전적 부동과 우연

더 정확한 표현은 이쪽이다.

> 생물 기관의 비효율은 국소최적의 엄밀한 증명이라기보다, 진화가 전역 설계가 아닌 **경로 의존적이고 제약된 점진적 탐색**이라는 강한 증거다.

---

## 18. 그래서 최종 비유는 이렇게 잡기

진화는 완전한 그리디가 아니다.

진화에는 돌연변이, 유전적 부동, 재조합, 개체군 다양성이 있다.

```text
그리디
- 후보 하나
- 현재 최선의 선택
- 보통 고정된 목적 함수

진화
- 여러 개체가 병렬로 탐색
- 무작위 변이와 재조합
- 선택은 확률적
- 환경이 바뀌면 적합도 함수도 바뀜
```

알고리즘으로 비유하면 이쪽에 더 가깝다.

```text
진화 ≈
population-based stochastic hill climbing
+ 가끔 나쁜 방향도 이동하는 random search
+ 계속 변하는 objective function
```

한 문장으로 기억하면:

> 진화는 그린필드 설계가 아니라, 스테이징과 장기 설계도 없이 프로덕션을 계속 운영하면서 이루어지는 확률적 점진 리팩터링이다.

---

## 🎯 나중에 다시 볼 포인트

- 유전 알고리즘은 진화에서 영감을 받은 탐색/최적화 알고리즘이다.
- 자연선택은 현재 환경에서 더 나은 변이를 고른다는 점에서 그리디와 닮았다.
- 그리디와 힐클라이밍처럼 진화도 지역 최적에 갇힐 수 있다.
- 생물 기관의 비효율은 국소최적의 엄밀한 증명이라기보다, 경로 의존적이고 제약된 점진 탐색의 증거로 보는 편이 정확하다.
- 진화는 개체군, 돌연변이, 유전적 부동, 재조합 덕분에 순수 그리디보다는 넓게 탐색한다.
- 진화는 exploit과 explore가 섞인 확률적 병렬 탐색으로 볼 수 있다.
- 자연 진화는 고정 목적 함수가 없고, 환경과 다른 생물도 같이 바뀌는 움직이는 최적화 문제에 가깝다.
- 백엔드 레거시 시스템처럼, 중간 단계가 살아남아야 최종 개선안까지 갈 수 있다.
