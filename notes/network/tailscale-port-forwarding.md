# Tailscale과 포트포워딩

> [!summary]
> Tailscale은 내 기기들끼리만 접속할 수 있는 개인 사설망을 쉽게 만들어주는 도구다.
>
> 홈서버나 맥북 터미널에 밖에서 접속하고 싶을 때, 공유기 포트포워딩 없이 안전한 접속 경로를 만들기 위해 많이 쓴다.

## 처음 헷갈렸던 질문

> [!question]
> 아이폰에서 맥북 터미널에 원격 접속하려고 했는데 왜 Tailscale을 깔라고 했을까?
>
> 그냥 아이폰에서 바로 맥북으로 원격 접속하면 안 되는 걸까?

아이폰에서 맥북 터미널에 접속하려면 아이폰이 맥북의 주소를 알아야 하고, 그 주소로 들어갈 길이 열려 있어야 한다.

같은 와이파이에 있으면 맥북의 내부 IP로 접속할 수 있다.

```text
아이폰
  |
같은 와이파이
  |
맥북
```

하지만 아이폰이 LTE/5G에 있고 맥북이 집 공유기 뒤에 있으면 이야기가 달라진다.

```text
아이폰 LTE/5G
  |
인터넷
  |
집 공유기
  |
맥북
```

이 상황에서 맥북은 인터넷에 직접 떠 있는 서버가 아니라 공유기 뒤에 있는 사설 기기다. 그래서 외부에서 그냥 `ssh 맥북`처럼 찾아갈 수 없다.

## Tailscale이 하는 일

Tailscale은 내 기기들끼리만 들어올 수 있는 사설 VPN 네트워크를 만들어준다.

예를 들어 맥북과 아이폰에 Tailscale을 설치하고 같은 계정으로 로그인하면, 둘은 같은 개인망에 들어온 것처럼 서로를 볼 수 있다.

```text
아이폰
  |
Tailscale 사설망
  |
맥북
```

이때 아이폰은 맥북의 Tailscale IP로 접속할 수 있다.

```bash
ssh mogi@100.x.x.x
```

여기서 `100.x.x.x`는 Tailscale이 맥북에 부여한 사설 IP다. 인터넷 전체에 공개된 주소가 아니라, 내 Tailscale 네트워크 안의 인증된 기기들이 쓰는 주소라고 보면 된다.

> [!info]
> Claude Code를 맥북 터미널에서 쓰는 상황이라면, Claude Code는 아이폰에서 실행되는 것이 아니다.
>
> 아이폰은 맥북 터미널을 조종하는 리모컨이고, 실제 Claude Code와 프로젝트 파일 수정은 맥북에서 일어난다.

## 홈서버에서 많이 쓰는 이유

홈서버는 보통 집 공유기 뒤에 있다.

```text
인터넷
  |
공유기
  |
홈서버
```

밖에서 홈서버에 접속하려면 원래는 공유기에 외부 접속 규칙을 만들어야 한다.

```text
외부 요청
-> 집 공유기
-> 홈서버
```

이때 필요한 설정이 보통 포트포워딩, DDNS, 방화벽, HTTPS 같은 것들이다.

Tailscale을 쓰면 홈서버를 인터넷 전체에 직접 공개하지 않고도 내 노트북, 아이폰, 서버끼리 접속할 수 있다.

개발자들이 홈서버에 Tailscale을 쓰는 이유는 다음과 같다.

- 포트포워딩 없이 밖에서 홈서버에 접속할 수 있다.
- 공유기 설정, 공인 IP, DDNS 문제를 덜 신경 써도 된다.
- SSH, Grafana, Jenkins, DB 같은 내부 도구를 인터넷에 직접 공개하지 않아도 된다.
- 맥북, 아이폰, 홈서버, 클라우드 서버를 같은 사설망처럼 묶을 수 있다.

## 포트포워딩이란

포트포워딩은 공유기한테 "밖에서 특정 포트로 들어오면, 집 안의 특정 기기로 보내줘"라고 규칙을 만드는 것이다.

집 안 기기들은 보통 사설 IP를 쓴다.

```text
맥북: 192.168.0.20
홈서버: 192.168.0.10
아이폰: 192.168.0.30
```

이 주소는 집 안에서만 통한다. 인터넷 밖에서는 `192.168.0.10`으로 집 안 홈서버를 찾아갈 수 없다.

밖에서 보이는 것은 보통 공유기의 공인 IP 하나다.

```text
집 밖에서 보이는 주소: 123.123.123.123
집 안 홈서버 주소: 192.168.0.10
```

그래서 공유기에 이런 규칙을 걸어둔다.

```text
123.123.123.123:8080
-> 192.168.0.10:8080
```

그러면 밖에서 공유기 8080번 포트로 들어온 요청이 홈서버의 8080번 포트로 전달된다.

SSH도 비슷하다.

```text
외부 포트 2222
-> 맥북 192.168.0.20:22
```

밖에서는 이렇게 접속한다.

```bash
ssh mogi@123.123.123.123 -p 2222
```

## 포트포워딩을 조심해야 하는 이유

포트포워딩은 집 안 서버로 들어오는 문을 인터넷 쪽에 열어두는 것이다.

예를 들어 다음처럼 열면 위험할 수 있다.

```text
외부 5432 -> 홈서버 PostgreSQL 5432
외부 6379 -> 홈서버 Redis 6379
외부 22   -> 맥북 SSH 22
```

이렇게 열려 있으면 인터넷의 아무나 해당 포트를 두드릴 수 있다. 비밀번호가 약하거나 설정이 잘못되어 있으면 실제 침입 경로가 될 수 있다.

> [!warning]
> DB, Redis, 관리자 페이지, SSH 같은 포트는 아무 생각 없이 인터넷에 열면 안 된다.
>
> 홈서버에서 내부 도구를 나만 쓰려는 목적이라면 Tailscale처럼 인증된 기기끼리만 접속하는 방식이 더 안전하고 편하다.

## Tailscale과 포트포워딩 비교

| 구분 | 포트포워딩 | Tailscale |
| --- | --- | --- |
| 접속 방식 | 공유기 공인 IP와 포트로 접속 | Tailscale 사설 IP로 접속 |
| 공개 범위 | 인터넷에서 포트가 보일 수 있음 | 내 Tailscale 네트워크 기기만 접근 |
| 설정 난이도 | 공유기, DDNS, 방화벽 설정 필요 | 각 기기에 설치하고 로그인 |
| 보안 감각 | 문을 열어두는 방식 | 인증된 기기끼리 통로를 만드는 방식 |
| 잘 맞는 상황 | 공개 웹서비스 운영 | 홈서버, 개인 SSH, 내부 도구 접속 |

> [!tip]
> 기억하기 쉽게 말하면:
>
> 포트포워딩은 공유기에 "이 문으로 들어오면 저 방으로 보내라"고 안내판을 붙이는 방식이다.
>
> Tailscale은 현관문은 그대로 두고, 내 기기들에게만 개인 통로를 만들어주는 방식이다.

## 백엔드 개발자 관점 예시

홈서버에 다음 서비스가 떠 있다고 하자.

```text
SSH: 22
Spring Boot: 8080
Grafana: 3000
PostgreSQL: 5432
Redis: 6379
```

이 포트들을 전부 인터넷에 공개하면 위험하다. 특히 PostgreSQL, Redis 같은 내부 저장소 포트는 외부에 열지 않는 것이 기본이다.

Tailscale을 쓰면 외부에서는 인터넷 공개 주소가 아니라 Tailscale 사설 IP로 접속한다.

```bash
ssh mogi@100.x.x.x
psql -h 100.x.x.x -p 5432
```

Grafana도 다음처럼 내 기기에서만 열 수 있다.

```text
http://100.x.x.x:3000
```

이 구조에서는 홈서버가 인터넷 전체에 공개된 서버라기보다, 내 기기들만 접근할 수 있는 개인 내부망 서버처럼 동작한다.

## 나중에 다시 볼 포인트

- 같은 와이파이에서는 내부 IP로 접속할 수 있지만, LTE/5G 같은 외부망에서는 보통 바로 접속할 수 없다.
- 외부에서 집 안 기기로 들어가려면 포트포워딩처럼 문을 열거나, Tailscale처럼 사설 통로를 만들어야 한다.
- 포트포워딩은 `외부 포트 -> 내부 IP:포트` 규칙이다.
- Tailscale은 원격 접속 앱 자체라기보다, 원격 접속이 가능하도록 안전한 네트워크 경로를 만들어주는 도구다.
- Claude Code를 아이폰에서 쓰는 경우에도 실제 실행은 맥북에서 되고, 아이폰은 맥북 터미널을 조작하는 역할이다.
